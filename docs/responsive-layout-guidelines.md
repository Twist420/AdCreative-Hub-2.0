# 响应式布局规范

## 适用范围

适用于需求中心、资产库、需求详情等业务后台页面。目标是浏览器缩小时仍保持信息层级清楚、操作入口稳定、文本不被挤压成竖排。

## 全局规则

- 页面主容器必须允许内容在可用宽度内收缩，外层布局使用 `min-w-0`。
- 顶部导航、筛选条、标签组可以横向滚动或自然换行，但不能撑出整页横向滚动。
- 卡片列表优先使用自动适配栅格，每张卡片设置最小可读宽度；空间不足时减少列数，而不是压缩卡片内部信息。
- 卡片内的标签组和操作组分开布局：标签在左侧自然换行，关键操作在右侧保持完整显示。
- 胶囊按钮、状态标签、优先级标签必须使用 `whitespace-nowrap`，避免中文被拆成竖排。
- 表单字段、日期、进度说明在窄卡片中可以降为单列；优先保证可读，不强行维持桌面宽布局。
- 底部主按钮和辅助信息不足以同排显示时，主按钮下移成独立一行。

## 协同看板卡片

- 卡片栅格采用 `auto-fit + minmax` 的思路，标准最小宽度约 `340px`。
- 顶部标签、优先级、编辑按钮不共享同一个强制单行容器。
- 负责人、场景、渠道、初版、截止属于元信息区，窄宽度下从两列降为一列。
- 需求 ID 和「新建需求」按钮在窄宽度下纵向排列，避免挤压 ID 列表。

## 验收口径

- `760px`、`900px`、`1100px` 桌面宽度下不出现整页横向滚动。
- 状态标签和操作按钮不出现单字竖排。
- 卡片之间不发生视觉重叠，操作按钮不跨到相邻卡片区域。
- 主要操作入口仍可见，且点击区域不小于当前规范画布中的按钮尺寸。
