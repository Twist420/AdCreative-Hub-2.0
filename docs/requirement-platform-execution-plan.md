# 需求平台执行计划

## 设计简报

这是广告创意与需求管理平台，不做营销页式的展示，而是做高频使用的业务后台。界面优先保证信息层级清楚、操作路径短、状态可追踪，并沿用 `components/UiSpecification.tsx` 的视觉规范。

当前开发优先跑通完整工作流：

1. 方向创建。
2. 方向下创建需求。
3. 需求继承方向字段并进入对应制作类型模板。
4. 需求审核。
5. 拆岗、评估难度并手动排期。
6. 制作人员推进任务状态。
7. 成品上传并沉淀资产。
8. 资产与投放数据回流到需求和方向。

## 开发原则

- 先闭环，后丰富：每次改动都尽量让一个真实工作流更完整，而不是只做单个静态页面。
- 保持业务后台密度：页面使用浅灰底承载内容区，白色卡片只用于独立对象，避免一个板子套一个板子。
- 操作层级固定：主动作必须明显，次动作靠近对应内容，危险动作收敛并二次确认。
- 字段口径统一：方向弹窗、需求大表、需求详情、协同看板使用同一套状态和字段含义。
- 手动优先：人员、时间、工时由组长或制作负责人手动填写，系统只提供人员占用、冲突和风险提示，不做一键自动改排。
- 模板分流：视频进入视频脚本模板；图片和试玩进入各自独立模板，但同样按小版本填写。

## 当前阶段

### 阶段一：需求中心闭环

已基本完成：

- 方向卡片点击打开方向详情。
- 方向下新建需求，并按方向制作类型分流。
- 需求详情修改状态后同步回协同看板。
- 视频、图片、试玩进入不同需求脚本界面。
- 需求大表默认以“新建需求”为最高优先级入口，筛选后显示列表。

继续补齐：

- 方向详情弹窗与需求大表字段继续保持一致。
- 需求编号、状态、进度按真实关联需求计算。
- 需求详情保存、删除和回写状态持续稳定。

### 阶段二：制作任务与排期

当前优先推进：

- 需求创建时自动生成默认制作任务。
- 任务支持岗位、前置关系、人员、开始时间、结束时间、预计工时。
- 人员、时间、工时由人工填写，系统只提示占用、冲突和截止风险。
- 任务状态驱动需求制作状态。
- 制作排期看板按任务粒度展示日历和甘特图，兼容旧需求级排期数据。
- 当前排期页聚焦手动排期辅助：查看人员占用、未来 7 天负荷和高优先级风险；人员排期情况通过弹窗查看，不在页面内铺开大日历。

下一步：

- 将未来一周排期健康度继续细化为可操作的风险列表。
- 增加最高/高优先级无法按截止日期完成的预警。
- 在需求详情手动排期时显示当前制作人的未来占用和日期冲突，减少来回切换看板。

### 阶段三：手动排期体验增强

先做前端可验证版本：

- 按岗位给出推荐人员分组。
- 选择人员后显示未来占用、冲突任务和可参考空档。
- 输入时间后提示是否撞期或超过方向截止日期。
- 输入人员后在看板中定位该人员当前排期。

需要暂停确认的决策：

- 飞书请假同步方式和权限。
- 中国节假日表的维护来源。
- 不同岗位的默认承接优先级和人员名单。

### 阶段四：资产与成品闭环

后续推进：

- 上传成品时关联需求、版本、尺寸、渠道和制作人。
- 成品上传后自动写回任务状态。
- 成品沉淀为资产库素材。
- 后续需求可以引用已沉淀资产，并追溯来源需求。

### 阶段五：数据回流

后续推进：

- 投放数据关联素材、需求和方向。
- 方向卡片展示转化结果和风险信号。
- 数据看板按周期、方向、渠道、制作类型筛选。

## 近期交付顺序

1. 完善需求详情右侧制作排期区，让任务可编辑难度、岗位、人员、时间和工时。

5. 再接入飞书请假与节假日配置，用于占用提示而不是自动改排。
